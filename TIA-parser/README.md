### Key parts

- `Program.cs`: CLI entry point and high-level orchestration.
- `src/cli`: argument parsing and usage output helpers.
- `src/parser`: parsing logic for TIA `.db` input files.
- `src/models`: data models shared across parser and generators.
- `src/generator`: code generation for `Sps.cs` and `SpsProxy.cs`.
- `input`: local test/sample input files.
- `output`: generated code output folder.
- `nupkg`: generated NuGet packages when packing as a .NET tool.
- `bin` and `obj`: build artifacts generated by the .NET SDK.

# Deployment

## As .exe

1. Run `dotnet publish -c Release -r win-x64 --self-contained true /p:PublishSingleFile=true` to deploy as an .exe
2. Take the .exe from `bin/Release/net10.0/win-x64/publish`

## As console tool

1. Run `dotnet pack -c Release -o .\nupkg`
2. You can use the TIA.parser.Tool file in the nupkg folder to install the tool
3. Run `dotnet tool install -g --add-source .\path\to\your\nupkg TIA.Parser.Tool` on your device to install the tool globally

### Uninstall

Uninstall using `dotnet tool uninstall --global TIA.Parser.Tool`
